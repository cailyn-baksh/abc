This project is a compiler system similar to LLVM, designed with esoteric languages in mind. The system includes a set of "front-ends" for several languages, and “back-ends” for several architectures. It also includes a standard intermediate representation (IR) virtual machine which allows the front ends and back ends to work in tandem without needing to be designed with the other specifically in mind. The general process of compiling is as follows:

  1. A compiler front-end (language) and back-end (architecture) are selected.
  2. The front-end performs first-stage compilation to IR
    1. The source file is tokenized by the front-end parser.
    2. The front-end performs first-pass optimization on the tokens.
    3. The first-stage compiler compiles the optimized tokens into IR.
  3. The optimizer optimizes the IR code in the primary optimization phase.
  4. The back-end compiler assembles IR code into machine code
    1. Each IR instruction is replaced with its machine code equivalent.
    2. The back-end performs peephole optimization.
    3. The resulting machine code is placed into an object file.
